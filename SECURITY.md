# Security

Please do not disclose receipt-parser or integrity-profile vulnerabilities in a
public issue before maintainers have had a chance to evaluate them.

Report a vulnerability through GitHub's private vulnerability reporting for
this repository. Include the affected version, a minimal receipt or raw JSON
reproduction, the expected integrity state, and the observed state. Never
include production credentials, private signing keys, or customer receipt data.

The contract separates content equality, cryptographic validity, and signer
trust. A conforming receipt or valid signature is not, by itself, evidence that
a signer should be trusted.
